# worlddashboard

このREADMEは、今後の新機能追加を**複数のAI Agentで進めるための共通ルール**を定義します。  
本プロジェクトでは「急がば回れ」の方針で、品質と再現性を優先して開発を進めます。

## 1. 目的
- AI Agentを活用した開発フローを標準化する
- 変更の品質・追跡性・安全性を確保する
- 最小変更で継続的に機能追加できる体制を作る

## 2. 役割
- **Director（統括）**
  - タスク分解、優先順位付け、最終判断を行う
  - 各Agentの成果を統合し、リリース可否を判断する
- **Implementation Agent（実装）**
  - 指定範囲のコード変更を実施する
  - 既存仕様を壊さない最小差分で対応する
- **Validation Agent（検証）**
  - テスト・ビルド・動作確認を実施する
  - 不具合を再現可能な形で報告する
- **Documentation Agent（文書化）**
  - READMEや仕様、運用ルールを更新する
  - 変更理由と運用手順を明確化する

## 3. 開発プロセス（新機能追加）
1. **要件整理**: Directorが目的・受け入れ条件を定義
2. **タスク分割**: 依存関係を考慮してAgentへ割り当て
3. **実装**: Implementation Agentが最小変更で対応
4. **検証**: Validation Agentが既存のビルド/テストを実行
5. **レビュー**: Directorが差分・影響範囲・セキュリティを確認
6. **文書化**: Documentation Agentが必要なドキュメントを更新
7. **統合**: Directorが最終承認しマージ判断

## 4. 作業ルール
- 変更は常に**小さく・明確に**する
- 無関係な修正は含めない
- 既存のlint/build/test手順を優先して使用する
- 変更内容は必ず記録し、再現可能な形で共有する
- セキュリティ上の懸念（依存関係・入力処理・秘密情報）を確認する

## 5. Definition of Done（完了条件）
- 要件を満たしている
- 既存の主要機能を壊していない
- 必要なテスト/ビルドが通っている
- 変更理由と運用上の注意がドキュメント化されている

## 6. コミュニケーション方針
- 事実ベースで報告する（実行コマンド、結果、差分）
- 不確実な点は仮定を明示する
- 判断に迷う場合はDirectorへエスカレーションする
